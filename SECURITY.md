# Security Policy

## Supported versions

Security fixes are applied to the latest released version.

## Reporting

Please report suspected vulnerabilities privately through GitHub's security advisory feature when available. Do not disclose exploitable details in a public issue before a fix is prepared.

## Data safety

This repository must not contain patient-level identifiable data, credentials, API keys, institutional secrets, or restricted datasets. Examples and tests must use synthetic or appropriately public data.

## Dependency security

Dependencies are intentionally limited. Proposed additions should explain why the dependency is needed and how it will be maintained.
